Например, дан пример кода для валидации XML-документа. Одну из вариаций можно записать в виде
класса ValidatorSAXwithXSD.

Данный код выполняет некоторую последовательность обязательных действий, определенных разработчиком библиотеки.
Чтобы решением можно было воспользоваться повторно, следует сделать фасад для него и фактически скрыть детали
реализации за фасадом метода: FacadeValidator.

Имена файлов и экземпляр класса-обработчика ошибок передаются в метод в качестве параметров. Метод же содержит
стандартную последовательность действий, вмешиваться в которую нет разумных оснований.
